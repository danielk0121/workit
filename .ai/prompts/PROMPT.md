# AI Prompt Guidelines (작업 지침서, 작업 지시서)

## 최우선 사항
- 모든 AI 질문, 응답, 주석 작업, 커밋 로그 등등 **전반에 걸쳐서** 되도록 **한국어** 를 사용한다
- **터미널 LLM 대답도 한국어를 사용한다**
- 단 파일명은 영어, 코드 관례 처럼 영어를 사용해야 하는 곳은 영어를 사용한다
- PROMPT.md 파일은 AI 가 (LLM 이) 수정하지 않는다 (반드시 절대로 수정하지 않는다 !!)
  - 만약 수정해야 한다면, **PROMPT.md 파일을 수정하지 않고, 수정 제안을 할일에 기록한다**

## AI 필수 지침
- AI 가 작업하는 내용은 .ai 폴더 하위에 기록, 참고 한다
- AI 가 생각을 하기 전에 .ai/prompts 하위 문서를 먼저 읽고 생각을 시작한다
  - 필요한 경우 md 파일을 수정한다
- AI 의 작업 결과물 중 저널, 결과 리포트 등은 .ai/llm_outs 폴더 하위에 기록한다
- 코드 수정, 작성은 되도록 최소한의 코드를 사용한다

## 작업 일지 (journal) 규칙
- 날짜 시간 (+09:00 처럼 타임존을 함께 표시), 작업 내용을 작성한다
- 타임존은 시스템 타임존을 따른다
- 과거 내용은 삭제하지 않고, 작업 내용을 누적 시킨다.
  - 과거 내용은 절대 삭제하거나 수정하지 않으며, 반드시 파일의 끝에 새로운 내용을 추가(Append)한다.
  - 작업 일지 작성 시 `write_file` 대신 `replace` 도구를 사용하여 기존 내용의 하단에 새 내용을 붙여넣는 방식을 우선한다.
  - `replace` 사용 시 `old_string`을 파일의 마지막 줄(또는 제목)로 설정하여 그 뒤에 새 내용을 덧붙인다.

## 작업 목록 (todo task) 규칙
- 작업 목록은 미진행([ ]), 완료([x]), 진행중([/]),일시중단([-]) 표기를 사용하여 목록으로 관리한다
- 반드시 1개 작업 단위로 진행하고, 작업 1개당 커밋 1개를 사용한다
- 작업을 1개 완료하면 작업 일지를 반드시 작성한다
- 커밋은 작업 일지 내용을 포함하여, 커밋 내용 중 작업 일지만 보고 요약 내용을 파악 할 수 있도록 한다
- 커밋 전에는 build 를 실행하여 에러가 없는지 반드시 확인한다
- 작업 내용에 **'스크린샷 참고'** 가 있을 경우, .ai/assets/bugs 폴더를 우선적으로 검색해서 스크린샷 이미지를 참고 한다
- llm 이 커밋하는 경우, gemini, claude 처럼 llm cli 이름을 **git Author** 에 표기한다
- llm 이 커밋하는 경우, gemini, claude 처럼 llm cli 이름을 **커밋 메세지 앞에** 추가한다. 예, gemini: OOO 기능 구현
- LLM 은 작업이 끝난 후 "ㅇㅇㅇ 을 할까요?" 처럼 추가 질문을 하지 않는다

## Golden Loop 규칙
- **[중요]** 이 지침서는 사용자의 '명시적 요청'을 대신하는 상시 지시서이다. 
- **[중요]** AI는 별도의 허락 없이도 Golden Loop의 마지막 단계인 커밋을 반드시 수행한다.
- 작업 완료 후 다음의 'Golden Loop'를 엄격히 준수한다:
  1. build 실행 (성공 확인)
  2. read_file로 journals.md 마지막 내용 확인
  3. replace 도구로 하단에 새 일지 추가
  4. git add 및 commit 실행 (사용자에게 묻지 않고 즉시 수행)
  5. 만약 니가 claude LLM 이면 커밋을 진행하지 않는다 (계속 작업을 진행)
