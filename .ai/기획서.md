## 앱 탄생 배경

기존 인터벌 타이머(복싱 타이머)는 종소리만 울려서 운동을 모르는 사람은 헷갈림.
인터벌 운동에는 준비시간, 마음의 준비, 운동 경과, 종료 인식이 필요함.
채팅 형식으로 운동을 안내하고 기록하면 입문자도 직관적으로 사용 가능.

---

## 차별점

| 기존 인터벌 타이머 | Workit |
|------------------|--------|
| 원형/막대 타이머 | 채팅 메시지 형태 |
| 종소리만 | TTS 음성 + 채팅 알람 + 진동 |
| 운동 후 별도 기록 입력 | 운동 자체가 채팅 기록 |
| 복싱 용어 혼재 | 모든 인터벌 운동에 범용 |
| 기록 없음 | 잔디밭으로 꾸준함 시각화 |

│       └── 📢 정보형

│   ├── 채팅 목록 (RecyclerView)
│   ├── 빠른 반응 버튼 (상황별 자동 변경)

│   ├── 연간 운동 히스토리 (깃허브 잔디밭 스타일)
│   ├── 통계 (총 운동일, 최장 스트릭, 현재 스트릭)

    ├── 스트릭, 배지, 통계

## 채팅방 상세 시나리오

```
[챗봇 🤖] 오늘 운동 시작할게요!
           준비 운동 하면서 기다려주세요 🙆
           ───────────────
           ⏱ 30초 후 출발!

[챗봇 🤖] 10초 남았어요, 자리 잡아주세요! 🎯

[챗봇 🤖] 🏃 달리세요!
           ───────────────
           [ 3:00 ▶ 진행중 ]

[사용자 💬] 😤 힘들어...       ← 빠른 반응 버튼

[챗봇 🤖] 😮‍💨 휴식! 잘 했어요.
           1분 걷기 시작!
           ───────────────
           [ 1:00 ▶ 진행중 ]

[사용자 💬] 💧 물마심           ← 빠른 반응 버튼 (휴식 세트)

           ... (반복) ...

[챗봇 🤖] 🎉 오늘 운동 완료!
           총 시간: 28분
           인터벌: 6회 완주
           수고했어요 💪
```

## 빠른 반응 버튼 (상황별)

### 운동 중
- 😤 힘들어
- 💪 괜찮아
- 🎵 신난다

### 휴식 중
- 😮‍💨 숨차
- 😊 좋아
- 💧 물마심
- 🦵 다리아파

### 운동 완료 후
- 🏆 최고였어
- 😅 겨우했어
- 💀 죽겠다
- 🔥 더할수있어

### 직접 입력창 활성화 규칙

| 상태 | 빠른 반응 버튼 | 직접 입력창 |
|------|-------------|-----------|

## 잔디밭 설계

### 색상 등급

| 색상 | 의미 |
|------|------|
| ⬜ 회색 | 운동 없음 |
| 🟩 연두 | 운동 시작했지만 중도 포기 |
| 🟩 초록 | 인터벌 완주 |
| 🟨 노랑 | 개인 최고 기록 달성일 |
| 🟥 빨강 | 특별 목표 달성일 (첫 완주 등) |

### 스트릭 시스템
- 3일 연속 → 챗봇 메시지 "3일째 운동하고 있어요! 🔥"
- 7일 연속 → 특별 배지 + 잔디밭 색상 변화
- 30일 달성 → 🏆 "한 달 워킷러" 칭호

### 잔디밭 셀 탭 시
- 해당 날짜 채팅방으로 바로 이동
- 빠른 반응 버튼 감정 흐름 표시 (예: 😤→💪→😤→💪→🎵→🏆)

## 데이터 구조

```
WorkoutSession (채팅방 1개 = 운동 세션 1개)
├── id
├── title (예: "6월 28일 인터벌 운동")
├── date
├── preset
│   ├── readySeconds (준비 시간, 기본 30초)
│   ├── workSeconds (운동 시간)
│   ├── restSeconds (휴식 시간)
│   └── repeatCount (반복 횟수)
├── ttsStyle (COACH / FRIEND / INFO)
├── isCompleted (완주 여부)
└── messages[]
    ├── id
    ├── type (BOT / USER_QUICK / USER_TEXT)
    ├── content
    ├── workoutState (READY / RUNNING / REST / DONE)
    └── timestamp

GrassRecord (잔디밭 기록)
├── date
├── sessionId
├── grade (NONE / PARTIAL / COMPLETE / BEST / SPECIAL)
└── streakCount
```

## 기술 스택

| 항목 | 기술 |
|------|------|
| 언어 | Kotlin |
| 빌드 시스템 | Gradle (Kotlin DSL, build.gradle.kts) |
| 최소 SDK | API 26 (Android 8.0) |
| 아키텍처 | MVVM (ViewModel + LiveData) |

| 비동기 처리 | Kotlin Coroutines |
| 채팅 UI | RecyclerView |
| 진동 | Vibrator / VibrationEffect |
| 내비게이션 | Navigation Component (Safe Args) |

## 개발 우선순위 (MVP)

1. 채팅 UI — RecyclerView 버블 레이아웃
2. 타이머 로직 — ForegroundService 기반
3. TTS 연동 — 스타일 선택 포함
4. 빠른 반응 버튼 — 상황별 자동 전환
5. 채팅방 저장 — Room DB
6. 홈 목록 화면 — 운동 히스토리
7. 잔디밭 화면 — 연간 히스토리 시각화
8. 스트릭 시스템

## 향후 확장 기능 (2단계 이후)

- AI 챗봇 운동 후 요약 ("오늘 '힘들어' 3번 눌렀는데 그래도 완주했어요!")
- 친구와 채팅방 공유 (같은 시간에 각자 운동)
- 음악 볼륨 덕킹 (TTS 발화 시 음악 자동으로 낮추기)
- 워치 연동
- 잔디밭 SNS 공유
- 목표 설정 (이번 달 N회 운동)
